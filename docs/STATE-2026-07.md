## The 6th halt & its fix (2026-06-30) — context for future divergence triage

`OverlayedDbProvider::succ` treated the `last` bound as inclusive; rippled's `ReadView::succ` is the
open interval `(key, last)`. Dormant for owner-dir walks; load-bearing for NFTokenMint page location
→ "NFT found in incorrect page" invariant divergence at #105284279 → 3-strike halt. Fixed in
`6893bbe` (+ mainline cherry `59bcaec`); regression test `overlay_succ_upper_bound_is_exclusive`.
Redeployed 2026-06-30 17:54; 38 NFTokenMints applied clean since, 0 new divergences.

## Roadmap (approved 2026-07-04)

Two parallel tracks toward a **fully-Rust validator running independently, FFI kept as backup +
differential oracle** (plan: `~/.claude/plans/golden-jumping-kurzweil.md` on .39):

1. **Track 1 — Lockstep M2 → sign-own-hash** (pacing): S1-S4 signature verification
   (proposals/validations/UNL-publisher/manifest rotation) → M2a in-round tx-set acquisition
   (gossip-first + `liTS_CANDIDATE` walk; real tx-set SHAMap root) → M2b CanonicalTXSet ordering +
   retry passes + skip-list (with a mutation-byte parity pre-soak) → M2c pseudo-txs through the
   in-round path (they ARRIVE in the consensus set — verified in RCLConsensus) → M2d shim TxMeta +
   close-time + full header → weeks of shadow → `XRPL_SIGN_SOURCE=own` (USER-GATED, phase-3 as
   instant fallback).
2. **Track 2 — pure-Rust engine** (background): new `crates/xrpl-engine` behind a `TxApplier`
   router (`XRPL_ENGINE=ffi|shadow|rust`), differential vs the FFI oracle per tx type, golden
   Number/STAmount vectors, va-08 forensic bundles + `replay_divergence`. Wave 1 = XRP-direct
   Payment, TrustSet, AccountSet, TicketCreate, DepositPreauth, SignerListSet, SetRegularKey.
3. **Track 3 — backup independence** (hardware-gated): NVMe + local xrpld on m3060;
   `XRPL_RPC_URLS=localhost,.39-failover`; multi-URL `XRPL_WS_URLS` support.
